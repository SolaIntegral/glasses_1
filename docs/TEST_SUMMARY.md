## 🎯 重要なテストシナリオ

### シナリオ1: 新規予約フロー
1. 講師が空き時間を登録
2. 生徒が講師詳細を閲覧
3. 生徒が予約（単発/定例選択）
4. 相談内容を入力
5. 予約確定
6. 講師にSlack通知
7. 予約一覧に表示（バッジ表示）

### シナリオ2: キャンセルフロー
1. 生徒が予約をキャンセル（24時間前）
2. 講師にキャンセル通知
3. 空き時間枠が復活

### シナリオ3: プロフィール管理フロー
1. 管理者が講師アカウント作成
2. 講師がプロフィール編集
3. 生徒がプロフィール閲覧
4. 全項目が正しく表示される

## 📋 テスト実行手順

### クイックテスト（15分）
1. 講師・生徒でログイン
2. プロフィール編集（最低限の項目）
3. 空き時間枠登録

5. Slack通知確認

### フルテスト（60分）
1. 事前準備
2. 必須機能すべて（約55項目）
3. 追加機能すべて（約8項目）
4. エラーログ確認

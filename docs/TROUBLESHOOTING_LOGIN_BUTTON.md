# ログインボタンが反応しない問題

## 🔍 症状

ログインボタンをクリックしても何も起こらない。コンソールにログが出力されない。

## 🐛 原因

### 想定される原因1: ブラウザキャッシュ

最も一般的な原因。古いHTMLファイルがブラウザにキャッシュされている可能性があります。

### 想定される原因2: ログインページが正しく表示されていない

URLが `/auth/login` なのに、ホームページのコンテンツが表示されている場合があります。

### 想定される原因3: JavaScriptが正しくロードされていない

静的エクスポートのNext.jsアプリケーションで、JavaScriptのハイドレーションが完了していない可能性があります。

## 🛠️ 解決方法

### ステップ1: ブラウザのキャッシュを完全にクリア

1. **Chrome/Edgeの場合**:
   - `F12` キーを押して開発者ツールを開く
   - ネットワークタブで右クリック → 「キャッシュを無効化」にチェック
   - 強制リロード: `Ctrl + F5` (Windows) または `Cmd + Shift + R` (Mac)

2. **Safariの場合**:
   - 環境設定 → 詳細 → 「メニューバーに"開発"メニューを表示」にチェック
   - 開発メニュー → 「キャッシュを空にする」
   - 強制リロード: `Cmd + Shift + R`

3. **手動で全キャッシュ削除**:
   - 設定 → 履歴をクリア → キャッシュされた画像とファイルを削除

### ステップ2: localStorageをクリア

ブラウザの開発者ツール → Consoleタブで以下を実行:

```javascript
localStorage.clear(); location.reload();
```

### ステップ3: シークレット/プライベートモードで試す

ブラウザのシークレット/プライベートモードで以下のURLを開く:
- `https://glasses1-582eb.web.app/auth/login`

### ステップ4: 異なるブラウザで試す

別のブラウザ（Chrome → Edge、Safari → Firefoxなど）で試してください。

### ステップ5: URLを直接入力

ログインページのURLを直接入力:
- `https://glasses1-582eb.web.app/auth/login`

## 🔍 デバッグ手順

### 1. コンソールでエラーを確認

開発者ツール（F12）→ Consoleタブを開き、以下のログが表示されるか確認:

```
Home page - loading: false user: null
```

これが表示されない場合は、ホームページが読み込まれています。

### 2. ログインページが正しく表示されているか確認

正しいログインページには以下が表示されるはずです:
- 中央に大きく「ログイン」というタイトル
- 「ユーザーID」と「パスワード」の入力欄
- 青い「ログイン」ボタン

これが表示されない場合、ホームページまたは別のページが表示されています。

### 3. ネットワークタブでリクエストを確認

開発者ツール → Networkタブ → ログインボタンをクリック → リクエストが送信されているか確認

## 🐛 よくある問題

### 問題1: 「ログイン」ボタンをクリックしてもページ遷移しない

**原因**: ホームページの「ログイン」ボタンは `/auth/login` へのリンクですが、実際のログインフォームが表示される `/auth/login` ページに移動している必要があります。

**確認方法**: 
- URLバーで `/auth/login` になっているか確認
- ログインフォーム（ユーザーID、パスワード入力欄）が表示されているか確認

### 問題2: ログインボタンをクリックしてもフォームが送信されない

**原因**: `onSubmit` ハンドラーが呼び出されていない可能性があります。

**確認方法**:
コンソールに以下のログが出力されるか確認:

```
handleSubmit called submit
preventDefault called
Attempting to login with userId: <userId>
```

これらが表示されない場合、フォームの `onSubmit` が発火していません。

## 📝 エスカレーション

上記の手順で解決しない場合、以下を収集して報告してください:

1. **ブラウザ情報**:
   - ブラウザ名とバージョン
   - デバイス（PC/モバイル）

2. **コンソールログ**:
   - 開発者ツール → Consoleタブの全ログ
   - 特にエラーメッセージ（赤文字）

3. **ネットワークログ**:
   - 開発者ツール → Networkタブ → ステータス400以上のリクエスト

4. **URL**:
   - 問題が発生している正確なURL
